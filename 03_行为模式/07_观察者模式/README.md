# 观察者模式 (Observer Pattern)

## 一、模式概述

**观察者模式**是一种行为型设计模式，它定义了对象之间的一对多依赖关系，当一个对象的状态发生变化时，所有依赖它的对象都会收到通知并自动更新。该模式通过解耦发布者（被观察对象）和订阅者（观察者），使系统更易于维护和扩展。

## 二、结构组成

| 角色 | 说明 |
|------|------|
| **抽象主题 (Subject)** | 定义添加、删除和通知观察者的接口（`NewsAgency`） |
| **具体主题 (Concrete Subject)** | 实现抽象主题接口，维护观察者列表并在状态变化时通知观察者（`Morgans`、`Gossip`） |
| **抽象观察者 (Observer)** | 定义接收通知的接口（`Observer`） |
| **具体观察者 (Concrete Observer)** | 实现抽象观察者接口，处理接收到的通知（`Dragon`、`Shanks`、`Bartolomeo`） |

## 三、海贼王场景示例

以《海贼王》中的新闻传播为例：摩根斯新闻社和八卦新闻社作为主题，革命军龙、红发香克斯、巴托洛米奥作为观察者订阅不同的新闻社。当新闻社发布新闻时，订阅者会收到相应通知。

- **Morgans (摩根斯新闻社)**: 正经新闻，有龙和香克斯两个订阅者
- **Gossip (八卦新闻社)**: 八卦新闻，有巴托洛米奥订阅
- **Dragon (龙)**: 路飞老爸，关注路飞成为四皇的消息
- **Shanks (香克斯)**: 路飞引路人，关注路飞消息
- **Bartolomeo (巴托洛米奥)**: 路飞头号粉丝，关注八卦

## 四、核心代码结构

```
include/
  ├── NewsAgency.hpp         # 抽象主题（前向声明 Observer）
  ├── Observer.hpp           # 抽象观察者（包含 NewsAgency.hpp）
  ├── ConcreteNews.hpp       # Morgans + Gossip 具体主题
  └── ConcreteObserver.hpp   # Dragon + Shanks + Bartolomeo 具体观察者
src/
  └── main.cpp               # 所有方法实现 + 测试入口
```

## 五、优缺点

### 优点
- **松耦合**: 主题和观察者之间抽象耦合
- **广播通信**: 一对多通知机制
- **符合开闭原则**: 增加新观察者无需修改主题

### 缺点
- **通知顺序不可控**: 多个观察者时的通知顺序不确定
- **可能引起循环依赖**: 观察者和主题之间相互调用
- **性能开销**: 观察者过多时通知效率降低

## 六、适用场景

- 一对多依赖关系，一个对象变化需要通知多个对象
- 发布-订阅模型、事件驱动系统
- GUI 事件处理、数据绑定
- 社交网络通知、消息推送
