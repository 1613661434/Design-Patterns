## 关键参与者

| 类 | 角色 | 职责 |
|----|------|------|
| `NewsAgency` | Subject (主题/被观察者) | 管理订阅者列表，定义通知接口 |
| `Morgans` | 具体主题 | 摩根斯新闻社 |
| `Gossip` | 具体主题 | 八卦新闻社 |
| `Observer` | Observer (观察者) | 定义更新接口，构造时自动订阅 |
| `Dragon/Shanks/Bartolomeo` | 具体观察者 | 收到新闻后的不同反应 |

## 调用流程

```
1. 创建主题: Morgans* ms = new Morgans
2. 创建观察者: Dragon* d = new Dragon("龙", ms) -> 构造时自动 ms->attach(d)
3. 通知: ms->notify("路飞成为四皇") -> 遍历 m_list -> 每个 item->update(msg)
4. 取消订阅: observer->unsubscribe() -> ms->deatch(obs)
```

## 何时使用

- 一对多依赖关系，一个对象变化需要通知多个对象
- 不希望观察者和被观察者之间紧密耦合
- 事件驱动系统、发布-订阅模型、GUI事件处理
- 社交网络通知、消息推送、数据绑定

## 编译与运行
