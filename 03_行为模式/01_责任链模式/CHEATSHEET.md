# 责任链模式 - 快速参考

## ASCII 结构图

```
┌──────────────────────────────────────────────────────┐
│                    Client (DaCongMing)                │
│                      发送请求                          │
└────────────────────────┬─────────────────────────────┘
                         │ request(type, manager)
                         ▼
┌──────────────────────────────────────────────────────┐
│              AbstractManager (抽象处理者)              │
│  + setNext(manager)    ← 设置后继                      │
│  + handleRequest(type) ← 纯虚函数                      │
│  # m_next: AbstractManager*                           │
└──────────────────────────────────────────────────────┘
                         △
         ┌───────────────┼───────────────┐
         │               │               │
         ▼               ▼               ▼
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│   Manager   │  │     CEO     │  │    Boss     │
│  部门经理    │──▶    CEO      │──▶   老板      │
│ (批请假)     │  │ (批请假+涨薪)│  │ (最终处理者) │
└─────────────┘  └─────────────┘  └─────────────┘
```

## 关键参与者

| 类 | 职责 | 处理能力 |
|----|------|----------|
| `AbstractManager` | 定义接口 `handleRequest()` 和 `setNext()` | -- |
| `Manager` | 部门经理，只能批请假 | QingJia |
| `CEO` | CEO，可批请假和涨薪 | QingJia, ZhangXin |
| `Boss` | 老板，所有请求都能处理 | 全部 |
| `DaCongMing` | 发送请求的员工 | -- |

## 调用流程

```
请假请求: Manager同意 -> 返回
涨薪请求: Manager -> CEO(给购物券) -> 返回
辞职请求: Manager -> CEO -> Boss(拒绝) -> 返回
```

## 何时使用

- 需要多个对象处理同一请求，但不希望发送者知道具体处理者
- 需要动态组合处理者（如审批流、中间件、过滤器链）
- 希望通过不同顺序的组合实现不同处理逻辑

## 编译与运行

```bash
make
./main
```
