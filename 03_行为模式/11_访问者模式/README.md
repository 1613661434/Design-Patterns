# 访问者模式 (Visitor Pattern)

## 一、模式概述

**访问者模式**是一种行为型设计模式，它允许将作用于某种数据结构中各元素的操作分离出来，形成独立的类，使系统在不修改数据结构的前提下，新增对元素的操作。该模式通过"双分派"机制，将数据结构与操作解耦，适用于数据结构稳定但操作频繁变化的场景。

## 二、结构组成

| 角色 | 说明 |
|------|------|
| **抽象访问者 (Visitor)** | 定义访问元素的接口（`AbstractAction`） |
| **具体访问者 (Concrete Visitor)** | 实现抽象访问者接口，定义对不同元素的操作（`Anger`、`Horror`） |
| **抽象元素 (Element)** | 定义接受访问者的接口（`AbstractMember`） |
| **具体元素 (Concrete Element)** | 实现抽象元素接口，接收访问者并调用其操作（`MaleMember`、`FemaleMember`） |
| **对象结构 (Object Structure)** | 管理元素集合，支持遍历和接受访问者（`CaoMaoTeam`） |

## 三、海贼王场景示例

以《海贼王》中草帽海贼团成员面对敌人时的反应为例：

- **MaleMember (男性成员)**: 路飞、索隆、山治、乔巴等
- **FemaleMember (女性成员)**: 娜美、罗宾
- **Anger (愤怒状态/访问者)**: 男性血战到底，女性叫大家快逃
- **Horror (恐惧状态/访问者)**: 男性辅助队友攻击，女性呼救

通过"双分派"机制，`accept(action)` 调用 `action->maleDoing(this)` 或 `action->femalDoing(this)`，根据成员性别和当前状态两个维度决定行为。

## 四、核心代码结构

```
include/
  ├── Visitor.hpp        # AbstractAction + Anger + Horror (前向声明成员类)
  ├── Member.hpp         # AbstractMember + MaleMember + FemaleMember
  └── CaoMaoTeam.hpp     # CaoMaoTeam 对象结构
src/
  └── main.cpp           # 所有方法实现 + 测试入口
```

## 五、优缺点

### 优点
- **开闭原则**: 增加新的访问操作很方便，无需修改元素类
- **单一职责**: 将相关操作集中在一个访问者类中
- **灵活性**: 可通过访问者对同一数据结构执行不同操作

### 缺点
- **元素变更困难**: 增加新元素类需要修改所有访问者
- **破坏封装**: 访问者可能需要访问元素的内部状态
- **复杂度高**: 双分派机制理解和使用都比较复杂

## 六、适用场景

- 数据结构（元素类）稳定，但需要频繁增加新操作
- 需要对不同类型元素执行不同操作，且需要避免类型判断
- 编译器设计（语法树遍历）、报表生成、数据统计等
