# 迭代器模式 (Iterator Pattern)

## 一、模式概述

**迭代器模式**是一种行为型设计模式，它提供一种顺序访问集合对象中元素的方式，而不暴露集合的底层表示。该模式将集合的遍历逻辑与集合本身解耦，使不同的遍历算法可以独立变化，客户端无需关心集合的具体实现。

## 二、结构组成

| 角色 | 说明 |
|------|------|
| **抽象迭代器 (Iterator)** | 定义遍历集合的接口（`first()`、`next()`、`isDone()`） |
| **具体迭代器 (Concrete Iterator)** | 实现抽象迭代器接口，维护遍历状态（`ForwardIterator`、`ReverseIterator`） |
| **抽象集合 (Aggregate)** | 定义创建迭代器的接口（`getIterator()`） |
| **具体集合 (Concrete Aggregate)** | 实现抽象集合接口，返回具体迭代器（`MyList`） |

## 三、海贼王场景示例

以《海贼王》中百兽海贼团的"凌空六子"及真打成员名单为例，使用双向链表存储成员列表，并通过迭代器实现正向和逆向遍历（如凯多检阅部下时的顺序访问）。

- **MyList**: 双向链表，存储海贼团成员名册
- **ForwardIterator**: 正向迭代器，从头到尾遍历
- **ReverseIterator**: 逆向迭代器，从尾到头遍历（凯多从后往前检阅）

## 四、核心代码结构

```
include/
  ├── NodeList.hpp       # Node 结构体 + MyList 双向链表
  └── Iterator.hpp       # Iterator + ForwardIterator + ReverseIterator
src/
  └── main.cpp           # 方法实现 + 测试入口
```

## 五、优缺点

### 优点
- **单一职责**: 将遍历逻辑与集合分离，各自独立变化
- **开闭原则**: 新增迭代器类型无需修改集合类
- **支持多种遍历方式**: 可以同时存在多个遍历状态
- **统一接口**: 对不同集合结构提供统一遍历方式

### 缺点
- **系统复杂度增加**: 需要额外的迭代器类
- **对简单集合可能是过度设计**: 简单数组用迭代器反而复杂

## 六、适用场景

- 需要为集合对象提供多种遍历方式
- 需要统一的遍历接口来遍历不同类型的集合
- 需要隐藏集合的底层实现（链表、树、图等）
- STL 迭代器理念的直接体现
