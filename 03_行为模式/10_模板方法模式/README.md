# 模板方法模式 (Template Method Pattern)

## 一、模式概述

**模板方法模式**是一种行为型设计模式，它在抽象类中定义了一个算法的骨架，将一些步骤延迟到子类中实现。该模式允许子类在不改变算法结构的情况下，重新定义算法中的某些步骤，从而实现代码复用和行为定制。

## 二、结构组成

| 角色 | 说明 |
|------|------|
| **抽象类 (Abstract Class)** | 定义模板方法（`getProperty()`），包含算法的骨架。声明抽象方法（`weapon()`、`appearance()`），由子类实现。提供具体方法（`canFlying()`、`isAuto()`），作为默认实现或钩子方法 |
| **具体子类 (Concrete Subclass)** | 实现抽象类中的抽象方法，定制算法的特定步骤。可选地覆盖钩子方法修改默认行为（`Pacifist`、`Seraphim`） |

## 三、海贼王场景示例

以《海贼王》中贝加庞克制造的机器人为例：

- **Pacifist (和平主义者)**: 基础机器人，能发射镭射光，无飞行能力
- **Seraphim (炽天使)**: 升级版机器人，继承和平主义者能力，增加飞行、自愈和更强的战斗能力

通过模板方法模式，将机器人属性的展示流程（`getProperty()`）固定在抽象类中，子类只需实现特定属性即可。

## 四、核心代码结构

```
include/
  └── Robot.hpp          # AbstractRobot + Pacifist + Seraphim
src/
  └── main.cpp           # 测试入口
```

## 五、优缺点

### 优点
- **代码复用**: 将公共代码放在抽象类中，避免子类重复
- **骨架固定**: 算法骨架在父类中定义，保证结构的统一性
- **易于扩展**: 子类只需实现特定步骤即可

### 缺点
- **子类数量增加**: 每种不同实现都需要一个子类
- **增加了复杂度**: 继承关系带来了额外的理解成本
- **里氏替换原则风险**: 子类可能改变父类预期行为

## 六、适用场景

- 多个子类有相同算法的骨架但具体步骤不同
- 需要控制子类的扩展流程（控制反转）
- 框架设计（如 Spring 的 JdbcTemplate）
- 构建工具、测试框架、数据处理流程等
