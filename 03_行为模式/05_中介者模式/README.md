# 中介者模式 (Mediator Pattern)

## 一、模式概述

**中介者模式**是一种行为型设计模式，它通过引入一个中介者对象来封装一系列对象之间的交互，使这些对象不再直接相互引用，而是通过中介者进行通信。该模式降低了对象间的耦合度，使系统更易于维护和扩展。

## 二、结构组成

| 角色 | 说明 |
|------|------|
| **抽象中介者 (Mediator)** | 定义中介者与同事类交互的接口（`MediatorOrg`） |
| **具体中介者 (Concrete Mediator)** | 实现抽象中介者接口，协调同事类的交互（`WorldGovt`、`GeMingArmy`） |
| **抽象同事类 (Colleague)** | 定义同事类与中介者交互的接口（`Country`） |
| **具体同事类 (Concrete Colleague)** | 实现抽象同事类接口，通过中介者与其他同事类交互（`Alabasta`、`Dressrosa`等） |

## 三、海贼王场景示例

以《海贼王》中的国家外交为例：不同国家通过中介机构进行消息传递。

- **WorldGovt (世界政府)**: 点对点中介 -- 消息仅发送给指定国家
- **GeMingArmy (革命军)**: 广播中介 -- 消息广播给所有成员国家
- **Alabasta (阿拉巴斯坦)**: 控诉德雷斯罗萨倒卖军火
- **Dressrosa (德雷斯罗萨)**: 反驳阿拉巴斯坦
- **Lulusia (露露西亚)**: 被伊姆毁灭，通过革命军广播
- **Kamabaka (卡玛巴卡)**: 革命军成员

## 四、核心代码结构

```
include/
  ├── Country.hpp        # 抽象国家类（前向声明 MediatorOrg）
  ├── MediatorOrg.hpp    # 抽象中介机构（前向声明 Country）
  ├── WorldGovt.hpp      # 世界政府（具体中介者）
  ├── GeMingArmy.hpp     # 革命军（具体中介者）
  ├── Alabasta.hpp       # 阿拉巴斯坦
  ├── Dressrosa.hpp      # 德雷斯罗萨
  ├── Lulusia.hpp        # 露露西亚王国
  └── Kamabaka.hpp       # 卡玛巴卡王国
src/
  └── main.cpp           # 所有方法实现 + 测试入口
```

## 五、优缺点

### 优点
- **降低耦合度**: 同事类之间不再直接依赖，只依赖中介者
- **集中控制**: 交互逻辑集中在中介者，易于维护
- **简化对象协议**: 多对多关系变为一对多关系

### 缺点
- **中介者可能过于庞大**: 集中了过多的交互逻辑，可能变成"上帝类"
- **复杂度转移**: 同事类简单了，但中介者变复杂了

## 六、适用场景

- 对象之间存在复杂的网状依赖关系
- 想通过一个中间类来封装多个类的行为，但又不想生成太多子类
- 多个对象之间有交互但逻辑混乱难以理解
- 聊天室、航空管制、GUI 组件交互等
