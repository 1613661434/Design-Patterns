# 享元模式 (Flyweight Pattern)

## 一、模式意图 (Intent)

通过**共享对象**来减少内存占用和提高性能，适用于需要创建大量相似对象的场景。该模式将对象的状态分为**内部状态**（可共享的公共数据）和**外部状态**（每个对象独有的数据），通过共享内部状态来避免重复创建对象。

## 二、模式结构 (Structure)

```
┌────────────────────┐
│   FlyweightBody    │  <-- Flyweight (抽象享元)
│   - m_sprite       │      内部状态: 精灵图
│   - m_color        │      内部状态: 渲染颜色
│   + move(x,y,speed)│
│   + draw(x,y)      │
└──────────┬─────────┘
           │
  ┌────────┼──────────────────┐
  │        │                  │
┌─┴──────────┴──┐    ┌───────┴──────────┐
│ SharedBombBody│    │   UniqueBomb     │
│ (共享享元)     │    │   (唯一享元)      │
│ 可共享复用     │    │   不共享,每例唯一   │
└───────────────┘    └──────────────────┘
        ^
        │ getSharedData(name)
┌───────┴──────────┐        ┌────────────────┐
│ BombBodyFactory  │───────>│   LaunchBomb   │  <-- Client
│ (享元工厂)        │ 提供    │   - m_x, m_y   │      外部状态
│ 管理共享对象池    │ 共享体  │   - m_speed    │      持有享元引用
└──────────────────┘        │   + move()     │
                            └────────────────┘
```

| 角色 | 类名 | 说明 |
|------|------|------|
| Flyweight（抽象享元） | FlyweightBody | 定义享元接口，包含内部状态 |
| ConcreteFlyweight（共享享元） | SharedBombBody | 可共享的炸弹弹体 |
| UniqueFlyweight（唯一享元） | UniqueBomb | 彩蛋，无需共享 |
| FlyweightFactory（享元工厂） | BombBodyFactory | 管理享元创建与共享 |
| Client（客户端） | LaunchBomb | 使用享元，维护外部状态 |

## 三、示例场景 (海贼王/游戏主题)

**游戏中的炸弹发射系统**: 游戏中需要发射大量炸弹，这些炸弹的弹体外观（精灵图、颜色 -- **内部状态**）可共享，而位置坐标、速度（**外部状态**）各不相同。享元模式通过共享弹体对象，减少内存消耗。

```
内部状态 (共享)          外部状态 (独立)
┌──────────────┐      ┌──────────────┐
│ 精灵图片      │      │ X坐标        │  ← 每个 LaunchBomb 不同
│ 渲染颜色      │      │ Y坐标        │
│ (弹体外观)    │      │ 速度         │
└──────────────┘      └──────────────┘
     共享                     独立
```

运行结果: 6个炸弹发射但只创建了3个弹体对象（"撒旦-1"复用1次, "撒旦-2"复用2次, "撒旦-3"新建）

## 四、优缺点

### 优点
1. **大幅减少内存**: 共享内部状态避免重复创建
2. **提高性能**: 减少对象创建和GC开销
3. **集中管理**: 工厂统一管理共享对象

### 缺点
1. **代码复杂**: 需要分离内部/外部状态
2. **线程安全**: 共享对象需要考虑线程安全问题
3. **外部状态开销**: 客户端需要维护大量外部状态

## 五、适用场景

- 系统中有大量相似对象，造成内存开销大
- 对象的大部分状态可以外部化
- 去除外部状态后，可以用较少共享对象代替大量对象
- 系统不依赖于对象标识（共享对象可被任意替换）

## 六、内部状态 vs 外部状态

| 特性 | 内部状态 | 外部状态 |
|------|----------|----------|
| 共享性 | 可共享 | 不可共享 |
| 存储位置 | 享元对象内部 | 客户端维护 |
| 变化性 | 相对稳定 | 经常变化 |
| 生命周期 | 与享元对象相同 | 由客户端管理 |
| 本例 | 精灵图(m_sprite), 颜色(m_color) | 坐标(x,y), 速度(speed) |
