# 简单工厂模式 (Simple Factory Pattern)

## 一、模式概述

**意图**：提供一种创建对象的方式，将对象的创建和使用分离。通过一个工厂类来负责创建产品对象，客户端只需通过参数向工厂类请求所需产品，而无需直接实例化对象。

**注意**：简单工厂模式并不属于 GoF 23种经典设计模式，但它是最基础、最常用的工厂模式变体。

**动机**：当客户端需要根据不同条件创建不同类型的产品时，如果直接使用 `new` 创建，客户端需要知道所有具体产品类的名字，导致耦合度高。简单工厂通过一个工厂类封装对象创建逻辑，实现解耦。

## 二、模式结构

```
                    +------------------+
                    |  AbstractSmile   |    (抽象产品)
                    |------------------|
                    | + transform()    |
                    | + ability()      |
                    +--------+---------+
                             |
            +----------------+----------------+
            |                |                |
    +-------+------+  +------+------+  +-----+-------+
    | SheepSmile   |  | LionSmile   |  | BatSmile    |  (具体产品)
    +--------------+  +-------------+  +-------------+
            ^                ^                ^
            |                |                |
            +--------+-------+-------+--------+
                     |               |
              +------+------+  enum Type
              | SmileFactory |  (SHEEP, LION, BAT)
              |--------------|
              | + createSmile(type): AbstractSmile*
              +--------------+
                     (工厂类)
```

## 三、核心参与者

| 角色 | 类名 | 说明 |
|------|------|------|
| 抽象产品 | `AbstractSmile` | 定义恶魔果实能力者的公共接口 |
| 具体产品 | `SheepSmile`, `LionSmile`, `BatSmile` | 实现不同形态的能力者 |
| 工厂类 | `SmileFactory` | 根据 `Type` 枚举创建对应产品 |
| 参数枚举 | `Type` | SHEEP, LION, BAT |

## 四、示例场景

以《海贼王》中的恶魔果实工厂为背景：
- 凯撒的工厂生产人造恶魔果实(SMILE)
- 不同果实赋予能力者不同的变身形态和特殊技能
- 客户端(买家)只需告诉工厂要哪种果实类型，工厂负责生产对应的能力者
- 绵羊果实: 变成山羊人，使用"巨羊角"
- 狮子果实: 变成狮子人，使用"火遁·豪火球之术"
- 蝙蝠果实: 变成蝙蝠人，使用"声纳引箭之万剑归宗"

## 五、优缺点

**优点**：
1. 实现了对象创建和使用的分离，客户端无需知道具体类名
2. 客户端只需传入参数即可获取所需对象
3. 工厂类集中管理对象创建逻辑，便于维护

**缺点**：
1. 工厂类职责过重，包含所有产品的创建逻辑
2. 新增产品时必须修改工厂类的 switch 语句，违反开闭原则
3. 工厂类通过 switch-case 判断，代码冗长

## 六、适用场景

- 需要创建的对象类型较少且相对固定
- 客户端不关心对象创建细节，只需产品接口
- 对象的创建逻辑相对简单

## 七、与工厂方法模式的对比

| 特性 | 简单工厂 | 工厂方法 |
|------|---------|---------|
| 工厂数量 | 1个 | 每产品1个 |
| 新增产品 | 修改工厂类 | 新增工厂子类 |
| 开闭原则 | 违反 | 遵循 |
| 复杂度 | 低 | 中 |

## 八、相关模式

- **工厂方法模式**：解决简单工厂违反开闭原则的问题
- **抽象工厂模式**：扩展到产品族的创建
