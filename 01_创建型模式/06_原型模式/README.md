# 原型模式 (Prototype Pattern)

## 一、模式概述

**意图**：通过复制现有对象（原型）来创建新对象，而无需通过传统的构造函数调用。

**别名**：克隆模式

**动机**：当对象的创建成本较高（如初始化耗时、资源密集、需要复杂配置）时，通过复制已有对象来创建新对象可以大幅提高性能。原型模式使用克隆机制代替 `new` 操作，特别适用于需要大量相似对象的场景。

## 二、模式结构

```
+----------------------------+
|       GermaSoldier         |   (抽象原型)
|----------------------------|
| + clone(): GermaSoldier*   |
| + whoAmI(): string         |
+------------+---------------+
             |
    +--------+--------+
    |                  |
+---+--------+  +------+------+
| Soldier66  |  | Soldier67   |  (具体原型)
|------------|  |-------------|
| + clone()  |  | + clone()   |
| + whoAmI() |  | + whoAmI()  |
+------------+  +-------------+
```

**核心参与者**：

| 角色 | 类名 | 说明 |
|------|------|------|
| 抽象原型 | `GermaSoldier` | 定义克隆自身的接口 `clone()` |
| 具体原型 | `Soldier66`, `Soldier67` | 实现克隆方法，复制自身实例 |

## 三、示例场景

以《海贼王》中杰尔马王国 (Germa Kingdom) 的超级士兵为背景：
- 杰尔马王国有两种超级士兵：Soldier66 和 Soldier67
- 通过克隆 (clone) 现有士兵来快速生产新士兵
- 每个克隆体拥有与原体相同的能力和属性
- 避免了每个士兵都从头初始化，节省时间和资源

## 四、克隆方式

### 浅拷贝 (Shallow Copy)
- 只复制对象本身的值
- 对指针成员，只复制指针地址，不复制指向的内容
- 可能造成多个对象共享同一块内存

### 深拷贝 (Deep Copy)
- 复制对象及其引用的所有对象
- 对指针成员，复制指针指向的内容
- 克隆后的对象与原对象完全独立

本例中使用的是**浅拷贝**（通过默认拷贝构造 `new Soldier66(*this)`），因为 Soldier66 和 Soldier67 没有指针成员，浅拷贝已经足够。

```cpp
GermaSoldier* clone() override
{
    return new Soldier66(*this);  // 调用拷贝构造函数
}
```

## 五、优缺点

**优点**：
1. 避免重复初始化，提高创建效率
2. 可以动态配置原型，在运行时改变创建对象的行为
3. 简化复杂对象的创建（无需知道具体类名）
4. 可以实现对象的保存和恢复（配合原型管理器）

**缺点**：
1. 每个类都需要实现 clone 方法，如果类层次深代码会重复
2. 含有循环引用的对象克隆较复杂
3. 深拷贝实现较为复杂，容易出错

## 六、适用场景

- 对象创建成本高（初始化复杂、数据库查询等）
- 需要创建大量相似对象（游戏中的敌人、子弹等）
- 需要保持对象的中间状态，并从该状态克隆
- 系统需要独立于产品的创建、组合和表示

## 七、相关模式

- **抽象工厂模式**：可以结合原型模式，用克隆代替工厂方法
- **备忘录模式**：克隆可以用于保存和恢复对象状态
- **组合模式 & 装饰模式**：使用这些模式的场景经常受益于原型
