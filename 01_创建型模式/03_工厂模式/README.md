# 工厂方法模式 (Factory Method Pattern)

## 一、模式概述

**意图**：定义一个创建对象的接口，但让子类决定实例化哪个类。工厂方法模式将对象的创建延迟到子类。

**别名**：虚拟构造器 (Virtual Constructor)

**动机**：简单工厂模式中，新增产品需要修改工厂类的 switch-case，违反开闭原则。工厂方法模式通过为每种产品创建专门的工厂子类，使系统在扩展新产品时无需修改已有代码。

## 二、模式结构

```
    +-------------------+           +------------------+
    | AbstractFactory   |           |  AbstractSmile   |  (抽象产品)
    |-------------------|           |------------------|
    | + createSmile()   |  creates  | + transform()    |
    +--------+----------+ --------> | + ability()      |
             |                      +--------+---------+
    +--------+---------+                    |
    |        |         |         +----------+----------+
    |        |         |         |          |          |
+---+--+ +---+--+ +---+--+  +---+----+ +---+----+ +---+----+
|Sheep | |Lion  | |Bat   |  | Sheep  | | Lion   | | Bat    |
|Fact  | |Fact  | |Fact  |  | Smile  | | Smile  | | Smile  |
+------+ +------+ +------+  +--------+ +--------+ +--------+
(具体工厂)                   (具体产品)
```

**核心参与者**：

| 角色 | 类名 | 说明 |
|------|------|------|
| 抽象产品 | `AbstractSmile` | 定义产品公共接口 (纯虚函数) |
| 具体产品 | `SheepSmile`, `LionSmile`, `BatSmile` | 实现产品接口 |
| 抽象工厂 | `AbstractFactory` | 声明创建产品的工厂方法 |
| 具体工厂 | `SheepFactory`, `LionFactory`, `BatFactory` | 实现工厂方法，创建对应产品 |

## 三、示例场景

以《海贼王》中扩展的恶魔果实生产系统为背景：
- 不同形态的恶魔果实需要专门的生产线来制造
- 绵羊果实工厂专门生产绵羊能力者
- 狮子果实工厂专门生产狮子能力者
- 蝙蝠果实工厂专门生产蝙蝠能力者
- 当需要新增果实形态（如毒蛇果实）时，只需新增 `SnakeSmile` 和 `SnakeFactory`，无需修改已有工厂代码

## 四、优缺点

**优点**：
1. 遵循开闭原则，新增产品无需修改已有代码
2. 客户端通过抽象工厂接口操作，符合依赖倒置原则
3. 工厂职责单一，每个工厂只负责一种产品

**缺点**：
1. 每个产品都需要一个对应的工厂类，类的数量成倍增加
2. 需要抽象出产品继承体系，增加了系统抽象性
3. 客户端需要选择具体的工厂类，一定程度上增加了使用复杂度

## 五、适用场景

- 客户端不知道所需对象的具体类
- 工厂类希望通过子类来指定创建哪个对象
- 需要支持产品的扩展，遵循开闭原则

## 六、与简单工厂的对比

| 维度 | 简单工厂 | 工厂方法 |
|------|---------|---------|
| 核心原则 | 封装创建逻辑 | 延迟到子类创建 |
| 工厂数量 | 1个 | N个 (每产品1个) |
| 新增产品 | 修改工厂类 (switch) | 新增子类 |
| 开闭原则 | 违反 | 遵循 |
| 类数量 | 少 | 多 |

## 七、相关模式

- **抽象工厂模式**：工厂方法通常被用于实现抽象工厂的具体工厂
- **模板方法模式**：工厂方法常与模板方法结合使用
- **原型模式**：创建对象的方式不同，原型通过克隆，工厂通过新建
